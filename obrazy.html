Rozdział 6: Obrazy

Osadzanie grafik, kontrola rozmiarów, dostępność oraz mapy odsyłaczy (map + area).

1. Znacznik <img>

Obraz wstawiamy za pomocą <img> - to element pusty, nie ma znacznika zamykającego.

<img src="zdjecie.jpg" alt="Opis zdjęcia" width="400" height="300" loading="lazy">
  • Atrybuty globalne: id, class, title, style itd.
  • Domyślnie jest elementem inline-block - można go stylizować CSS.

2. Atrybut src

src (source) określa ścieżkę do pliku graficznego. Obsługiwane formaty: JPG, PNG, GIF, WEBP, SVG, AVIF.

<img src="obrazy/logo.png">                <!-- względny - podfolder -->
<img src="../assets/baner.jpg">            <!-- katalog wyżej -->
<img src="https://example.com/foto.jpg">   <!-- bezwzględny URL -->
W projektach lokalnych używaj ścieżek względnych. Zewnętrzne URL-e mogą być blokowane przez hotlinkowanie.

3. Atrybut alt

alt wyświetla się gdy obraz się nie załaduje i jest odczytywany przez czytniki ekranu.

<img src="kot.jpg" alt="Bury kot leżący na parapecie">
<img src="ikona.png" alt="">   <!-- puste alt dla obrazów dekoracyjnych -->
  • Zawsze dodawaj alt - wymagane przez WCAG.
  • Dla obrazów w linkach alt opisuje cel linku.
  • Puste alt="" jest poprawne gdy obraz nie wnosi treści.

4. width i height

Atrybuty width i height określają wymiary w pikselach. Pomagają przeglądarce zarezerwować miejsce i uniknąć przeskoków układu.

<img src="baner.jpg" width="800" height="400" alt="Baner">
width / height w HTML - wymiary bezwzględne w px. Jeśli nie zachowane proporcje, obraz będzie zniekształcony.
CSS width / height - elastyczne jednostki. Najlepszy sposób: img { max-width: 100%; height: auto; }.
Ustaw width i height w znaczniku dla stabilności układu, a w CSS użyj max-width:100%; height:auto; dla responsywności.

5. Obraz jako link

Otocz <img> kotwicą <a> żeby cały obraz był klikalny.

<a href="galeria.html">
    <img src="miniaturka.jpg" alt="Miniatura galerii" width="150">
</a>
  • Domyślnie wokół obrazka pojawia się niebieska ramka. Usuń przez CSS: a img { border: none; }.
  • Zadbaj o opisowy alt - określa cel linku.

6. Mapa odsyłaczy - wprowadzenie

Mapa odsyłaczy pozwala zdefiniować wiele klikalnych obszarów na jednym obrazku, każdy prowadzący do innego adresu. Wymaga trzech elementów: <img>, <map> i <area>.

  • Przydatna przy planach biur, mapach regionów, schematach technicznych.
  • Można łączyć różne kształty (prostokąty, okręgi, wielokąty).
  • Uwaga: współrzędne są bezwzględne - nie skalują się automatycznie z obrazem.

7. Atrybut usemap

usemap dodajemy do <img>. Wartość musi zaczynać się od # i odpowiadać atrybutowi name w <map>.

<img src="plan.png" alt="Plan biura" usemap="#biuroMap">
Składnia: usemap="#nazwaMapy" - znak # jest obowiązkowy.

8. Znacznik <map>

<map> definiuje mapę odsyłaczy. Wymaga atrybutu name.

<map name="biuroMap" id="biuroMap">

</map>

Wewnątrz <map> umieszczamy znaczniki <area>.

9. Znacznik <area>

Pojedynczy klikalny obszar definiujemy przez <area />. Atrybuty:

AtrybutOpisPrzykład

coordsWspółrzędne zależne od kształtu, liczby oddzielone przecinkami.coords="10,10,80,120"
hrefCel linkuhref="pokoj.html"
altOpis alternatywny - wymagany dla dostępnościalt="Pokój 101"
targetOkno docelowe: _blank, _selftarget="_blank"
titlePodpowiedź po najechaniutitle="Sala konferencyjna"

10. shape i coords - składnia

Współrzędne podajemy jako ciąg liczb oddzielonych przecinkami. Punkt (0,0) to lewy górny narożnik obrazka.

shapecoords - znaczeniePrzykład
rectx1,y1,x2,y2 (lewy górny i prawy dolny róg)coords="20,20,120,80"
circlex,y,radius (środek i promień)coords="200,150,50"
polyx1,y1,x2,y2,x3,y3,... (wierzchołki wielokąta)coords="100,200,200,250,50,250"
defaultcały obrazek, bez współrzędnychshape="default" href="strona.html"
Jak zdobyć współrzędne? Otwórz obraz w programie graficznym (GIMP, Photoshop) i odczytaj pozycję kursora. Są też generatory map online.

Dla poly minimalna liczba to 3 pary (trójkąt). Więcej punktów = bardziej złożony wielokąt.

11. Działający przykład mapy odsyłaczy

Poniżej działający przykład. Kliknięcie na każdy obszar przenosi do innej kotwicy na tej samej stronie.

Kliknij na zielony prostokąt, żółte koło lub pomarańczowy trójkąt.

<img src="plan.svg" usemap="#demoMapa" alt="Plan pomieszczeń">
<map name="demoMapa">
    <area shape="rect" coords="20,20,120,80" href="recepcja.html" alt="Recepcja">
    <area shape="circle" coords="250,80,35" href="sala.html" alt="Sala A">
    <area shape="poly" coords="100,200,200,250,50,250" href="garaz.html" alt="Garaż">
</map>

Wyjaśnienie współrzędnych:

  • Prostokąt: coords="20,20,120,80" - lewy górny (20,20), prawy dolny (120,80).
  • Okrąg: coords="250,80,35" - środek (250,80), promień 35px.
  • Wielokąt: coords="100,200,200,250,50,250" - trzy wierzchołki trójkąta.

12. Dodatkowe kwestie map odsyłaczy

Responsywność - współrzędne są stałe, więc skalowanie obrazu CSS rozjeżdża obszary. Nowoczesne podejście to interaktywne SVG lub nakładki CSS.
Kolejność area - można nakładać kształty, przeglądarka obsługuje pierwszy pasujący obszar. Kolejność ma znaczenie.
Dostępność - każdy <area> musi mieć alt. Klawisz Tab przechodzi przez obszary.
Istnieją edytory map online (wystarczy wpisać "image map generator"). Rysujesz obszary na obrazie i dostajesz gotowy kod HTML.

14. Podsumowanie

  • Zawsze dodawaj alt - dostępność i SEO.
  • Używaj width i height żeby uniknąć skoków układu.
  • Nowoczesne formaty: WebP lub AVIF mają lepszą kompresję niż JPG/PNG.
  • Mapy odsyłaczy sprawdzają się przy schematach ze stałymi współrzędnymi.
  • Opisuj każdy <area> atrybutem alt i title.

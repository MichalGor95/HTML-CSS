Rozdział 1: Wprowadzenie do HTML

Czym jest HTML, jak zbudowana jest strona, czym są znaczniki i atrybuty oraz dlaczego <!DOCTYPE html> jest ważny.

1. Czym jest HTML?

HTML (HyperText Markup Language) to język znaczników używany do tworzenia struktury i treści stron internetowych. Definiuje co jest nagłówkiem, akapitem, obrazkiem czy linkiem.

  • HyperText - możliwość tworzenia odnośników prowadzących do innych dokumentów.
  • Markup - znaczniki otaczające treść i nadające jej znaczenie semantyczne.
  • HTML nie jest językiem programowania - nie wykonuje obliczeń ani logiki, tylko opisuje układ dokumentu.
Wersje HTML: od HTML 2.0 (1995) przez HTML 4.01, XHTML, aż do HTML5 (standard od 2014). Obecnie używamy HTML5 - wprowadza nowe znaczniki jak <article>, <section>, <nav>. 
<h1>Witaj na mojej stronie</h1>
<p>HTML to podstawa każdej witryny.</p>

2. Struktura dokumentu HTML

Każda poprawna strona HTML ma określoną hierarchię. Najważniejsze elementy to:

<!DOCTYPE html>
<html lang="pl">
  <head>
    <meta charset="UTF-8">
    <title>Tytuł strony</title>
  </head>
  <body>
    <h1>Główny nagłówek</h1>
    <p>Treść widoczna dla użytkownika.</p>
  </body>
</html>
<html> - główny kontener całego dokumentu. Atrybut lang określa język strony.
<head> - metadane: tytuł, kodowanie, linki do stylów - niewidoczna na stronie.
<body> - cała widoczna treść: teksty, obrazy, przyciski.
<title> - tytuł wyświetlany na karcie przeglądarki, ważny dla SEO.
Bez odpowiedniej struktury przeglądarka może wyświetlić treść, ale strona będzie niepoprawna semantycznie. Zawsze używaj <html>, <head> i <body>.

3. Znaczniki (tagi)

Znaczniki to słowa kluczowe w nawiasach ostrych < >. Większość występuje w parach: otwierający <tag> i zamykający </tag>. W środku jest treść.

<h1>To jest nagłówek pierwszego stopnia</h1>
<p>To jest akapit tekstu.</p>

Są też znaczniki puste (self-closing), które nie mają zamknięcia ani treści wewnętrznej:

  • <img src="foto.jpg" alt="Opis"> - obraz
  • <br> - przełamanie linii
  • <hr> - pozioma linia
  • <input type="text"> - pole formularza
W HTML5 nie trzeba zamykać pustych znaczników ukośnikiem (<br> zamiast <br />), ale składnia XML-owa też jest dozwolona.

Znaczniki blokowe i liniowe

TypPrzykładyCharakterystyka
Blokowe<div>, <p>, <h1>-<h6>, <section>, <article>Zajmują całą szerokość, zaczynają nową linię.
Liniowe (inline)<span>, <a>, <strong>, <em>, <img>Nie łamią linii, zajmują tyle miejsca ile potrzebują.

4. Atrybuty

Atrybuty umieszczamy w znaczniku otwierającym w postaci nazwa="wartość". Nadają elementom dodatkowe właściwości.

<a href="https://example.com" target="_blank">Kliknij mnie</a>
<img src="logo.png" alt="Logo firmy" width="150">
<input type="text" id="imie" placeholder="Wpisz imię">
href - cel linku src - źródło obrazu alt - tekst alternatywny width/height - wymiary id - unikalny identyfikator class - grupowanie elementów style - styl inline title - podpowiedź

Poprawnie

<a href="strona.html">Więcej</a>
<img src="foto.jpg" alt="Opis">
Atrybuty zawsze w cudzysłowie, nazwy małymi literami.

Błędnie

<a href=strona.html>Więcej</a> - brak cudzysłowu
<img SRC="foto.jpg"> - wielkie litery, niezalecane

5. <!DOCTYPE html>

Deklaracja DOCTYPE informuje przeglądarkę, której wersji HTML użyć do renderowania strony. W HTML5 wygląda tak:

<!DOCTYPE html>
  • Musi być pierwszą linią dokumentu, przed <html>.
  • Nie jest znacznikiem HTML, tylko instrukcją dla przeglądarki.
  • Brak DOCTYPE może włączyć tryb quirks - nieprzewidywalne renderowanie.
  • W HTML5 jest case-insensitive, ale konwencja to pisanie wielkimi literami.
W HTML 4.01 deklaracja DOCTYPE była długa i zawierała odwołanie do DTD. HTML5 to uprościł do jednej linii.

6. Minimalna strona HTML

<!DOCTYPE html>
<html lang="pl">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Moja pierwsza strona</title>
</head>
<body>
    <h1>Witaj w HTML!</h1>
    <p>To jest akapit tekstu.</p>
    <a href="https://www.w3schools.com">Odwiedź poradnik</a>
</body>
</html>
  • <!DOCTYPE html> - standard HTML5.
  • <html lang="pl"> - główny kontener, język polski.
  • <head> - metadane: kodowanie UTF-8, viewport, tytuł.
  • <body> - widoczna treść: nagłówek, akapit i link.
Atrybut viewport jest ważny dla stron mobilnych - bez niego strona może wyglądać jak na desktopie, pomniejszona.

7. Zagnieżdżanie elementów

Znaczniki mogą być umieszczane wewnątrz innych. Ważne jest poprawne domknięcie - ostatni otwarty musi być pierwszy zamknięty.

<div>
    <p>To jest <strong>ważny</strong> akapit.</p>
</div>

Błąd zagnieżdżenia:

<p>Tekst <strong>niewłaściwie</p></strong> - nieprawidłowa kolejność zamykania.
<p>Tekst <strong>poprawnie</strong></p> - prawidłowo.

8. Komentarze

Komentarze są ignorowane przez przeglądarkę. Służą do opisywania kodu.

<!-- To jest komentarz - niewidoczny na stronie -->
  • Komentarze mogą być wieloliniowe.
  • Nie można ich zagnieżdżać.
  • Komentuj złożone fragmenty, ale bez przesady.

9. Co musisz zapamiętać

HTML = struktura + treść - nie style, nie logika.
Otwierający i zamykający - poprawne zagnieżdżanie.
Atrybuty - nazwa="wartość", zawsze w cudzysłowie.
<!DOCTYPE html> - zawsze na początku.
Lang - atrybut lang w <html> pomaga SEO i czytnikom ekranu.
